# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

All commands run from `react-app/` directory:

```bash
npm run dev      # dev server (Vite)
npm run build    # TypeScript check + Vite build
npm run lint     # ESLint
npm run preview  # preview production build
```

There are no tests. The build (`tsc -b && vite build`) is the primary correctness check — TypeScript errors will fail CI.

## Architecture

**Stack:** React 19, TypeScript, Vite, Tailwind CSS v4, Supabase (auth + DB + storage), TanStack Query, Zustand, React Router v7 (HashRouter), Sonner toasts, dnd-kit, Three.js / React Three Fiber.

**Deployment:** GitHub Pages via `.github/workflows/deploy.yml`. Env vars `VITE_SUPABASE_URL` and `VITE_SUPABASE_ANON` are GitHub secrets.

### Auth & permissions

- `src/stores/authStore.ts` — Zustand store holding `user` (Supabase auth), `profile` (from `profiles` table), and `isAdmin()`.
- `isAdmin()` returns `true` when `profile.role === 'admin'`.
- Login uses username → Supabase RPC `get_email_by_username` → `signInWithPassword`.
- `AuthGuard` / `AdminGuard` components in `App.tsx` protect routes.
- Per-task permissions in `ProjectPage`: `canFullEdit = admin || isCreator`, `canChangeStatus = admin || isCreator || isAssigned`, `canDelete = admin || isCreator`.

### Data fetching

All server state uses **TanStack Query** (`useQuery`, `useQueryClient`). Global `staleTime` is 30 s. After mutations, always call `queryClient.invalidateQueries({ queryKey: [...] })` — never mutate cache directly.

Realtime updates use Supabase channels (`supabase.channel(...).on('postgres_changes', ...).subscribe()`). These invalidate the same query keys so the UI stays in sync.

### Page structure

Large pages are split into `src/pages/PageName/` folders with `index.tsx` as the entry point:

- `src/pages/ProjectPage/` — main task management page. `index.tsx` holds `ProjectPage`, sub-components in separate files (`TaskDetailModal`, `CreateTaskModal`, `SortableTaskRow`, `TaskGroup`, `ManageSubprojectsModal`, `EditProjectModal`, `BulkCreateTasksModal`). Shared CSS class `inputClass` and small utilities live in `shared.ts`.
- `src/pages/ModelsPage/` — 3D models list page (upload, thumbnails, project assignment). The actual 3D viewer lives in `src/viewer-core/` (see below); `viewerAdapter.ts` here implements its Supabase persistence, `shared.ts` exports `BUCKET`.

### 3D viewer core (`src/viewer-core/`)

Pure 3D viewer shared between the main app and the standalone sandbox `3d-lab/` (sibling of `react-app/`). **Must stay free of app imports** — no `@/lib/supabase`, no `@/stores`, no `@/components`. Persistence and permissions are injected via props on `Viewer`: `adapter: ViewerAdapter` (data CRUD — Supabase impl in `ModelsPage/viewerAdapter.ts`, localStorage impl in `3d-lab/src/localAdapter.ts`), `canEdit: boolean`, `confirm: ConfirmFn`.

Files: `Viewer.tsx` (full-screen viewer UI), `ModelWithReveal.tsx` (GLTF loading + reveal animation), `CameraControls.tsx` (fly cam, presets, persist), `Vegetation.tsx`, `ViewerTools.tsx` (measure, screenshot), `AnnotationMarkers.tsx`, `Environment3D.tsx` (SunSky + shadows, procedural GroundPlane, SectionPlane via `gl.clippingPlanes`, CameraFlyTo, N8AO/Bloom Effects), `adapter.ts` (ViewerAdapter interface + types), `shared.ts` (`SceneNode`, `CameraState`, `setMeshGlow`), `index.ts` (barrel — import via `@/viewer-core`).

Viewer features beyond basic viewing: environment panel (studio/outdoor sun with time-of-day, ground island, post-fx), section plane, saved camera views + presentation mode (drag-reorder + rename; each view can have assigned annotations that stagger-fade in once the camera arrives and fade out before the next view — `SavedView.annotationIds`, migrations `sql/004` sort_order + `sql/005` annotation_ids), object renaming and user layers in the scene panel, and a global PBR material library (`materials.ts` + `MaterialsPanel.tsx`): textures auto-detected from filename tokens (_albedo, _normal, _roughness…), converted client-side to WebP max 2048² (normal maps kept untouched), UVs generated by world-space box projection with per-assignment tile size (meters)/rotation/offset. Material defs live in Supabase table `viewer_materials` (migration `sql/003_viewer_materials.sql`) with textures in the `models` bucket under `textures/{materialId}/`; per-object assignments persist in `SceneOrg.materials`. The lab stores texture blobs in IndexedDB. Renames/layers/env settings persist as one JSON blob (`SceneOrg`) via `adapter.fetchSceneOrg`/`saveSceneOrg`; saved views via `fetchViews`/`createView`/`deleteView`. Supabase tables `model_scene_org` and `model_views` — migration in `sql/002_viewer_scene_org_views.sql` (adapter degrades gracefully if tables are missing: reads return empty, writes toast an error). Renames are display-only overrides keyed by original `mesh.name` — colors and vegetation keep targeting original names.

The `3d-lab/` sandbox consumes these files directly via Vite alias `@core` + `resolve.dedupe` — changes here show up live in both apps. To work on the viewer in isolation: `cd 3d-lab && npm run dev` (drag&drop a `.glb`, no login/Supabase needed).

Single-file pages: `DashboardPage`, `MyTasksPage`, `ReportsPage`, `ThreeDMaxPage`, `ReviewPage`, `InspiracePage`.

### Key data model notes

- **Task assignees** use two fields: `tasks.assigned_to` (legacy single user) and `task_assignees` table (multi-user). Both are checked when evaluating `isAssigned`. When self-assigning, only `task_assignees` is updated — `assigned_to` is not changed.
- **`selectedTask` in ProjectPage** is derived from `tasks.find(t => t.id === selectedTaskId)`, not stored as a snapshot, so it always reflects the latest query data.
- **3D model data** (annotations, vegetation) links to `model_files.id`. Re-uploading a model file keeps the same `id`, preserving all linked data.

### UI components (`src/components/ui/`)

- `Modal` — wraps Radix Dialog, sizes: `sm | md | lg | xl`
- `Button` — variants: `primary | secondary | danger | ghost`, sizes: `sm | md`
- `StatusBadge` / `PriorityBadge` — read-only colored chips
- `InlineSelect` / `InlineDateInput` — click-to-edit cells in task table
- `useConfirm()` — imperative confirm dialog, returns a Promise
- `Avatar` — colored initials avatar, prop `small` for 24px size

### Supabase storage buckets

- `models` — 3D model files (`.glb`/`.gltf`) and their thumbnails (`thumbs/`)
- `attachments` — task file attachments (`task-files/<task_id>/`) and comment images (`comment-images/<task_id>/`)
